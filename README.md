## 1. 요약

본 프로젝트는 네이버 IT/과학 및 경제 뉴스를 매일 자동으로 수집하고, OpenAI로 요약한 뒤 카카오톡으로 전송하는 개인화 뉴스 자동화 봇이다.

사용자의 핵심 문제는 **정보 수집의 비효율과 분야 단편화**다. 기존 카카오 오픈채팅 뉴스봇은 IT/과학 분야만 제공하며, 경제 뉴스를 함께 확인하려면 별도로 검색해야 한다. 또한 매일 직접 뉴스 사이트에 접속해 중요 기사를 판단하는 과정은 반복적이고 시간 소모적이다.

이 봇은 IT/과학 + 경제 두 분야를 통합 크롤링하고, 단일 OpenAI API 호출로 요약을 생성해 카카오톡으로 전송함으로써 이 문제를 해결한다.

핵심 차별점은 두 가지다. 첫째, 기존 오픈채팅 봇이 제공하지 않는 **경제 뉴스를 IT 뉴스와 함께 제공**한다. 둘째, 두 분야를 **단일 API 호출**로 처리해 OpenAI API 토큰 비용 외 추가 비용이 발생하지 않는다.

## 2. 제품 개요

### 2.1 비전

매일 아침 카카오톡 하나로 IT·경제 트렌드를 동시에 파악한다.

### 2.2 제품 가설

IT 직군을 목표로 하는 취준생 및 개발자는 IT 뉴스와 경제 뉴스를 모두 챙겨야 하지만, 두 분야를 각각 확인하는 루틴을 유지하기 어렵다. 기존 오픈채팅 봇은 IT/과학만 제공해 경제 뉴스는 별도로 검색해야 한다. 두 분야를 하나의 카카오톡 메시지로 통합하면 이 불편을 해소할 수 있다.

### 2.3 핵심 가치 제안

- IT/과학 + 경제 두 분야를 하나의 메시지로 통합 제공한다.
- 기존 오픈채팅 봇 대비 경제 뉴스라는 추가 가치를 제공한다.
- AI 요약으로 기사 제목만이 아닌 핵심 맥락을 함께 전달한다.
- 두 분야를 단일 API 호출로 처리해 OpenAI API 토큰 비용만 발생한다.
- GitHub Actions 기반으로 서버 비용 없이 운영 가능하다.

## 3. 문제 정의

뉴스 소비와 IT·경제 트렌드 파악은 현재 비효율적인 워크플로우다.

### 3.1 사용자 불편 사항

1. 매일 직접 네이버 뉴스 IT 섹션과 경제 섹션을 각각 접속해야 한다.
2. 수십 개의 기사 중 중요한 것을 직접 판단해야 한다.
3. 기존 오픈채팅 뉴스봇은 IT/과학 분야만 제공하며 경제 뉴스가 없다.
4. 여러 뉴스봇을 구독하면 채팅방이 분산되어 관리가 불편하다.

### 3.2 기존 도구의 한계

- 네이버 뉴스 앱은 알림이 많고 원하는 카테고리만 받기 어렵다.
- RSS 리더는 설정이 번거롭고 매일 능동적으로 확인해야 한다.
- 카카오 오픈채팅 뉴스봇은 IT/과학만 제공, 커스터마이징 불가, 알고리즘 불투명하다.

## 4. 목표와 비목표

### 4.1 제품 목표

1. 매일 오전 8시에 네이버 IT/과학(sid1=105) + 경제(sid1=101) 뉴스를 각 5건씩 자동 수집해야 한다.
2. 수집된 두 분야 뉴스를 **단일** OpenAI **API 호출**로 함께 요약해야 한다.
3. IT 섹션과 경제 섹션을 구분해 하나의 카카오톡 메시지로 전송해야 한다.
4. GitHub Actions 기반으로 별도 서버 없이 자동 실행되어야 한다.
5. API 키 등 민감 정보는 환경변수로 관리해야 한다.

### 4.2 MVP 비목표

1. 여러 사람에게 동시에 전송하는 단체 발송 기능
2. 사용자 맞춤 키워드 필터링 기능
3. 웹 UI 또는 별도 앱 개발
4. 뉴스 히스토리 저장 및 검색 기능
5. IT/경제 외 다른 카테고리 지원

### 6.2 핵심 파이프라인

```
GitHub Actions 스케줄 트리거 (매일 오전 8시 KST)
  → Spring Boot 애플리케이션 실행
  → NaverNewsCrawler: IT/과학 5건 + 경제 5건 순차 수집
  → NewsSummarizer: 두 분야 합쳐서 단일 OpenAI API 호출로 요약 생성
  → KakaoMessageSender: Access Token 갱신 후 IT/경제 섹션 구분 메시지 전송
  → 실행 완료 로그 출력
```

| 경제 뉴스 크롤링 | Jsoup으로 네이버 경제 뉴스 5건 수집 | P0 |
| 단일 AI 요약 호출 | 두 분야를 하나의 Claude API 요청으로 처리 | P0 |

| 토큰 자동 갱신 | Refresh Token으로 Access Token 재발급 | P0 |
| 스케줄 자동 실행 | GitHub Actions cron으로 매일 자동 실행 | P0 |

## 7. 제품 경험

### 7.1 UX 원칙

1. **Zero-Action**: 사용자는 아무것도 하지 않아도 매일 메시지가 도착한다.
2. **Scannable**: 카카오톡 메시지 하나로 IT + 경제 10개 뉴스와 요약을 빠르게 훑을 수 있다.
3. **Separated**: IT와 경제 섹션을 명확히 구분해 원하는 분야를 바로 찾을 수 있다.
4. **Linkable**: 더 읽고 싶은 기사는 링크를 클릭해 바로 이동할 수 있다.

### 7.2 카카오톡 메시지 포맷

```
📰 오늘의 뉴스 브리핑 (03/29)

💻 IT/과학
1. AI가 슈퍼마리오 게임을 한다고?...광주과기원 국제대회 우승
   https://n.news.naver.com/...
2. 호주 이어 인도네시아도 청소년 소셜미디어 금지
   https://n.news.naver.com/...
...

📈 경제
1. 미국 관세 우려에 코스피 0.8% 하락
   https://n.news.naver.com/...
2. 한국은행, 기준금리 동결 결정
   https://n.news.naver.com/...

🤖 AI 요약

[IT/과학]
1. [AI 게임] 광주과기원이 AI 게임 플레이 국제대회 우승, 국내 AI 연구 수준 입증

[경제]
1. [증시] 미국 관세 우려로 코스피 하락, 외국인 매도세 지속
2. [금리] 한국은행 기준금리 동결, 경기 불확실성 반영
```

## 8. 기능 요구사항

### 8.1 뉴스 크롤링

- 시스템은 네이버 뉴스 IT/과학(sid1=105)과 경제(sid1=101) 섹션에서 각 5건씩 수집해야 한다.
- 시스템은 각 기사의 제목과 링크를 함께 수집해야 한다.
- 시스템은 분야별로 중복 기사를 제거해야 한다.
- 한 분야 크롤링 실패 시 나머지 분야만으로 이후 단계를 계속 진행해야 한다.

### 8.2 AI 요약 (단일 호출)

- 시스템은 IT/과학 5건 + 경제 5건을 **하나의** OpenAI **API 요청**에 담아 전송해야 한다.
- 프롬프트는 분야 구분을 명시해 IT 요약 5건과 경제 요약 5건을 함께 반환받아야 한다.
- API 호출은 하루 1회로 제한해 불필요한 토큰 사용을 방지해야 한다.
- API 호출 실패 시 예외를 기록하고 제목만 전송하는 fallback으로 전환해야 한다.

### 8.3 카카오톡 전송

- 시스템은 매 전송 전 Refresh Token으로 Access Token을 갱신해야 한다.
- 메시지는 IT 섹션과 경제 섹션을 구분해 구성해야 한다.
- 메시지는 2000자를 초과하지 않도록 사전 검증해야 한다.
- 전송 성공/실패 여부를 로그로 출력해야 한다.

### 8.4 자동 실행

- GitHub Actions cron은 매일 UTC 23:00 (KST 08:00)에 실행되어야 한다.
- workflow_dispatch로 수동 실행도 가능해야 한다.
- ANTHROPIC_API_KEY, KAKAO_CLIENT_ID, KAKAO_REFRESH_TOKEN은 GitHub Secrets로 관리해야 한다.
